Git Commands and Description

  • git init : Initalizing the git in the Project.

  • git clone [url] : Retrieving the Source code from the given github url

  • git status : Shows modified files (commits) in the working directory

  • git add [file or files] : Add a file or files to git (to commit) it will be in stage.

  • git reset [file] : Unstagimg a file while retaining the changes in working directory.

  • git reset --hard [commit id] : clear staging area, Rewrite working tree from specific commit.

  • git reset --soft [commit id] : clear staging area, But we can see the commit in the logs

  • git commit -m [msg] : Commiting the staged conntent as new commit.

  • git branch [branch_name] : Creating a new branch form the main branch.

  • git checkout : Switching to other branch.

  • git merge : To merge the specific branch source code to other branch.

  • git log : We can see the logs of commits with id, Author and datetime.

  • git fetch : To view all the branches of the repository.

  • git pull : Fetch source code from commits from the tracking main branch.

  • git push : merge source code from commits from the tracking main branch to currently working branch.
